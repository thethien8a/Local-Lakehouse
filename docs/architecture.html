Medallion Architecture

Dữ liệu đi qua 3 tầng xử lý, mỗi tầng tăng chất lượng và giảm độ phức tạp

📥 Landing Zone
Raw Parquet
→
🥉 Bronze
Raw Ingestion
→
🥈 Silver
Clean + Enrich
→
🥇 Gold
Star Schema
→
📊 Dashboard
Metabase / Trino

Bronze Layer

nessie.taxi.bronze

Mục đích

  • Ingest dữ liệu thô từ Landing Zone (Parquet files)
  • Thêm cột ingestion_date để partition
  • Idempotent — bỏ qua ngày đã ingest

Đặc điểm

  • Append-only (không transform)
  • Partition theo ingestion_date
  • Format: Parquet trên Iceberg

Silver Layer

nessie.taxi.silver

Mục đích

  • Làm sạch: loại null, lọc ID hợp lệ, loại giá trị âm
  • Feature engineering: trip_id, trip_duration, avg_speed...
  • WAP Pattern: tạo branch → validate → merge

Data Quality Rules

  • VendorID ∈ {1,2,6,7}
  • RatecodeID ∈ {1,2,3,4,5,6,99}
  • payment_type ∈ {0,1,2,3,4,5,6}
  • fare_amount, total_amount, trip_distance > 0
  • pickup ≤ dropoff

Gold Layer

Star Schema

Mục đích

  • Tạo Star Schema phục vụ dashboard
  • Fact table: fact_trips
  • Dim tables: dim_zone, dim_payment, dim_date
  • Aggregate: agg_daily_summary

WAP Pattern

  • Tạo branch từ main
  • Write data vào branch
  • Merge branch về main khi thành công
