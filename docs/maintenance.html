Maintenance

Iceberg Table Maintenance

Bảo trì định kỳ giúp tối ưu hiệu năng đọc/ghi và giải phóng dung lượng lưu trữ

Compaction

Mục đích

  • Gộp small files thành larger files
  • Tăng hiệu năng đọc (ít file hơn = ít seek hơn)
  • Chạy hàng tuần (Chủ nhật 2:00 AM)

Cơ chế

  • rewriteDataFiles() gộp files < 128MB
  • Target file size: 128MB
  • Chỉ gộp, không xóa data

Expire Snapshots

Mục đích

  • Xóa snapshot history cũ để tiết kiệm metadata
  • Giữ lại 10 snapshot gần nhất
  • Chạy hàng tháng (ngày 1, 3:00 AM)

Cơ chế

  • expireSnapshots() xóa snapshot > 30 ngày
  • retainLast = 10 (luôn giữ 10 snapshot mới)
  • Giải phóng metadata file size

Remove Orphan Files

Mục đích

  • Xóa data files không còn được tham chiếu bởi bất kỳ snapshot nào
  • Giải phóng dung lượng lưu trữ thực tế
  • Chạy hàng tháng (sau expire snapshots)

Cơ chế

  • removeOrphanFiles() quét và xóa file mồ côi
  • Chỉ chạy sau khi expire snapshots xong
  • Thư mục data thực sự được giải phóng
